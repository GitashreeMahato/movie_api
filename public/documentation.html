MyFlix Application

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Business Logic URL HTTP Method Request body data format Response body data format
Get a list of movies /movies GET None A JSON object holding data about all the movies
Get data about a single movie by title /movies/:title GET None A JSON object holding data about a single movie, containing description, genre, director, image URL, and whether it’s featured or not property. Example: { "Genres": { "_id": "652172ea0e38b2a356c819b5" }, "Directors": { "_id": "65217a050e38b2a356c819ba" }, "_id": "652196100e38b2a356c819dd", "Title": "Inception", "Description": "A thief, who enters people's dreams to steal their secrets, is given a final job where he must implant an idea into someone's mind.", "Actors": [ { "_id": "65218e380e38b2a356c819c5" }, { "_id": "65218e400e38b2a356c819c6" }, { "_id": "65218e480e38b2a356c819c7" } ], "Release_date": 2010, "Rating": 8.8, "imageURL": "https://www.imdb.com/title/tt1375666/mediaviewer/rm3426651392/?ref_=tt_ov_i", "Featured": true }
Get data about genre by name /movies/genre/:genreName GET None A JSON object holding data about a genre containing name and description properties. Get all users /users GET None A JSON object holding data about all movies, structured like this: [ { "_id": "652596d0ef5ff34d917bd889", "username": "Trisha", "password": "tri@123", "email": "trisha@gmail.com'", "birth_date": "1994-09-09T22:00:00.000Z", "favorite_movies": [], "__v": 0 }, { "_id": "6527179651263fcda50740c7", "username": "Ricky Saha", "password": "rick@123%567", "email": "rickySaha@gmail.com", "birth_date": "1998-09-22T00:00:00.000Z", "favorite_movies": [ "6523c4ee0e38b2a356c819df", "6523ca990e38b2a356c819e3" ], "__v": 0 } ]
Create a new User /users POST A JSON object holding data about a user to add , structured like: { "username": "Trisha", "password": "tri@123", "email": "trisha@gmail.com'", "birth_date": "1994/09/09" } A JSON object holding data about a user that was added, inluding an ID: { "_id": "652596d0ef5ff34d917bd889", "username": "Trisha", "password": "tri@123", "email": "trisha@gmail.com'", "birth_date": "1994-09-09T22:00:00.000Z", "favorite_movies": [], "__v": 0 }
Login a user /login POST A JSON object holding user data, structured like this: { "password": "rick@123%567", "email": "rickySaha@gmail.com" } A JSON object holding data about the user logs an application and creates a JWT, structured like this: { "user": { "_id": "6527179651263fcda50740c7", "username": "Ricky Saha", "password": "rick@123", "email": "ricky@gmail.com'", "birth_date": "1998-09-22T00:00:00.000Z", "favorite_movies": [], "__v": 0 }, "token": [JWT Bearer Token String] }
Update a user's info (name, password,email, birth_date) /users/:username PUT A JSON object holding user data, structured like this: { "password": "rick@123%567", "email": "rickySaha@gmail.com" } A JSON object holding data about a user that was updated: { "_id": "6527179651263fcda50740c7", "username": "Ricky Saha", "password": "rick@123%567", "email": "rickySaha@gmail.com", "birth_date": "1998-09-22T00:00:00.000Z", "favorite_movies": [ "6523c4ee0e38b2a356c819df", "6523ca990e38b2a356c819e3" ], "__v": 0 }
